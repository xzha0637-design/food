# 食物拍照评分 App 设计文档

## 概述

一个拍照识别食物并评分的移动端 PWA，个人自用。

## 架构

纯前端 PWA，单 HTML 文件 + JSON 数据库 + localStorage，托管 GitHub Pages。

```
手机浏览器 (PWA)
  ├─ 拍照/选图 → Claude Vision API → 返回食物名称+份量
  ├─ 食物名称 → 本地评分数据库 (food_db.json) → 返回分数
  ├─ 份量 → 本地营养素数据库 (nutrients.json) → 计算营养平衡
  └─ 确认保存 → localStorage
```

## 数据流

1. 用户拍照 → 图片转 base64
2. 发送给 Claude Vision API，prompt 要求识别食物名称、估算每样食物重量(g)
3. 匹配 food_db.json 获取综合质量分
4. 匹配 nutrients.json 获取营养素数据
5. 计算：
   - 综合质量分 = 各食物评分的克重加权平均
   - 营养平衡分 = 实际营养素比例 vs 膳食指南推荐比例的匹配度
   - 一顿饭得分 = 综合质量分 × 0.7 + 营养平衡分 × 0.3
6. 展示结果，用户确认后存入 localStorage

## 评分公式

- 综合质量分 = Σ(food_score_i × grams_i) / Σ(grams_i)
- 营养平衡分 = 100 - 各营养素偏差的扣分
  - 热量偏差(±30%范围内不扣分，超出按比例扣)
  - 三大营养素比例偏差
  - 钠超标扣分
- 一顿饭得分 = 综合质量分 × 0.7 + 营养平衡分 × 0.3
- 一天得分 = 当天所有食物的总克重加权平均
- 一周得分 = 7天得分的平均

## 营养素参考标准（成年男性）

| 营养素 | 每日推荐 | 每餐参考(1/3) |
|--------|---------|-------------|
| 热量 | 2250 kcal | 750 kcal |
| 蛋白质 | 65g | 22g |
| 碳水 | 325g | 108g |
| 脂肪 | 60g | 20g |
| 膳食纤维 | 28g | 9g |
| 钠 | <6000mg | <2000mg |

## 页面

1. **拍照页（主页）**：拍照按钮 + 今日食物列表 + 今日评分
2. **结果页**：AI 识别结果、份量、得分明细、营养评价、确认保存
3. **统计页**：今日评分、本周趋势图、营养素周均值对比

## 技术选型

- 视觉 AI：Claude Vision (Haiku)
- 前端：单 HTML + vanilla JS + PWA manifest
- 图表：Chart.js CDN
- 托管：GitHub Pages
- 存储：localStorage

## 数据文件

- `food_db.json`：食物综合评分，~150 条目
- `nutrients.json`：食物营养素(每100g)，~250 条目

## 预估时间

数据准备 0.5 天 + 核心功能 0.5 天 + UI 0.5 天 + 统计 0.5 天 = 2 天
